# Нейросеть для предсказания калорийности блюд

Проект по построению мультимодальной нейросетевой модели для предсказания калорийности блюд.

Согласно постановке задачи, модель должна оценивать калорийность блюда на основе доступной информации и потенциально может использоваться в fitness- и health-приложениях для автоматического подсчёта калорий.

Для предсказания используются три источника данных:

- изображение блюда;
- список ингредиентов;
- общая масса блюда.

Задача формулируется как **регрессия**, где целевой переменной является общая калорийность блюда.

## Подход к решению

Калорийность блюда определяется не одним фактором.

Список ингредиентов позволяет понять состав блюда, изображение содержит визуальную информацию о размере порции и пропорциях компонентов, а масса напрямую связана с общим количеством продукта.

Поэтому был выбран мультимодальный подход, объединяющий все три источника информации.

Предварительные эксперименты показали, что использование нескольких модальностей действительно улучшает качество модели:

| Конфигурация                | MAE |
|-----------------------------|-----|
| Текст + масса               | ~45 |
| Изображение + масса         | ~60 |
| Текст + изображение + масса | ~33 |

Наиболее информативной оказалась текстовая модальность, однако добавление визуальных признаков дало заметное улучшение итогового качества.

## Архитектура модели

Используется мультимодальная архитектура с late fusion.

### Текстовая ветка

Для обработки списка ингредиентов используется BERT (bert-base-uncased).

Текст преобразуется в токены, затем через BERT извлекаются эмбеддинги, после чего применяется mean pooling для получения единого векторного представления блюда.

### Визуальная ветка

Для обработки изображений используется EfficientNet-B0 с предобученными весами ImageNet.

Верхние слои модели частично размораживаются для дообучения под конкретную задачу.

### Числовая ветка

Масса блюда подаётся как отдельный числовой признак и проходит через линейную проекцию в общее пространство признаков.

### Fusion

Признаки всех модальностей приводятся к общей размерности, объединяются и передаются в регрессионную голову.

Схема модели:

```text
Ингредиенты ───> BERT ───────────────┐
                                     │
Изображение ──> EfficientNet ────────┼──> Fusion ──> Regressor ──> Калорийность
                                     │
Масса ─────────> Linear projection ──┘
```

## Анализ ошибок и ограничения

Наибольшие ошибки чаще возникают на блюдах с высокой калорийной плотностью:

* орехи;
* сыр;
* пицца;
* блюда с большим количеством масла;
* жирные мясные блюда.

Основное ограничение задачи — отсутствие информации о количестве отдельных ингредиентов.

Например, запись `almonds` сообщает модели о наличии орехов, но не позволяет понять, идёт ли речь о 10, 50 или 150 граммах продукта.

Изображение частично компенсирует этот недостаток, но по RGB-фото также невозможно точно определить массу каждого компонента.

Минимальные ошибки чаще наблюдаются у блюд:

* с простым составом;
* с одним доминирующим ингредиентом;
* с визуально очевидной структурой.
